# RoadRunner PHP SDK

RoadRunner SDK monorepo for PHP. The code of five interop packages (`Worker`,
`Http`, `KeyValue`, `Services`, `Lock`) is copied into `src/` under a single
`RoadRunner\Sdk\<Module>` namespace and is being rewritten for the new
RoadRunner API (interop over gRPC, `spiral/grpc-client`; the old goridge
transport is being removed). Protobuf DTOs come from the local path package
`packages/roadrunner-api` (`roadrunner-php/roadrunner-api-dto`).

- PSR-4: `RoadRunner\Sdk\ => src/`, `RoadRunner\Tests\ => tests/`.
- PHP >= 8.2.

## Protobuf / gRPC DTOs

DTOs and gRPC service interfaces are **generated**, not hand-written. The
generated package `packages/roadrunner-api/` (`roadrunner-php/roadrunner-api-dto`)
is wired in as a local path repository and is **untracked in git** ŌĆö regenerate
it rather than editing files by hand. `packages/temporal-api/` is regenerated by
the same script.

### Sources

- `.proto` definitions live in the **`proto/` git submodule**
  (`roadrunner-server/api`, see `.gitmodules`). Run `git submodule update --init`
  if `proto/` is empty. RoadRunner protos are under `proto/roadrunner/api/`,
  vendored google/temporal protos under `proto/third_party/`.
- The new gRPC API is **V2**: e.g. the HTTP service is
  `proto/roadrunner/api/http/v2/{service,request,response,http}.proto` ŌåÆ
  `RoadRunner\HTTP\DTO\V2\HttpProxyServiceInterface` + DTOs.

### Regenerating

```bash
scripts/generate.sh        # regenerates both roadrunner-api and temporal-api
composer dump-autoload     # pick up new/removed classes
```

`scripts/generate.sh` (bash; runs under Git Bash on Windows) per package:

1. `buf build <proto-module>` resolves all imports (incl. well-known types) into
   a single `FileDescriptorSet` under `runtime/` (`*.binpb`). buf is used only
   for dependency resolution ŌĆö no BSR plugins.
2. `protoc --descriptor_set_in=<fds> --php_out --php-grpc_out` emits both message
   DTOs and `*Interface extends Spiral\RoadRunner\GRPC\ServiceInterface` stubs in
   one call (`protoc-gen-php-grpc` plugin).
3. Wipes `packages/<pkg>/src` and moves the `...\DTO\V<n>\` namespace leaves into
   `src/<short-path>/`, stripping redundant `RoadRunner/` / `DTO/` segments.
4. Calls `scripts/scaffold-packages.sh`, which rewrites each package's
   `composer.json` PSR-4 map from the on-disk layout (one entry per `V<n>` leaf,
   e.g. `RoadRunner\HTTP\DTO\ => src/HTTP/`).

### Tooling

`buf`, `protoc`, and `protoc-gen-php-grpc` are fetched into `runtime/` by
`vendor/bin/dload get` (see `dload.xml`). If they're already in `runtime/`
(`*.exe` on Windows) the script reuses them. Override discovery via
`$BUF` / `$PROTOC` / `$PROTOC_GEN_PHP_GRPC`.

### Constructing DTOs

Use **fluent setters**, not the array constructor:

```php
$response = (new HttpHandlerResponse())
    ->setId($request->getId())
    ->setStatus(200)
    ->setBody($body)
    ->setHeaders([
        'Content-Type' => (new HttpHeaderValue())->setValues(['text/plain']),
    ]);
```

Setters carry explicit per-field typing (`int|string` for `int64`, `MapField`
for maps, etc.) so misuse fails at the call site instead of silently producing
a default-valued message on the wire.

## Tests

Framework ŌĆö **Testo** (not PHPUnit). Running:

```bash
composer test            # XDEBUG_MODE=coverage + testo
vendor/bin/testo         # add --no-coverage to skip coverage
vendor/bin/testo --suite=Worker/Unit
```

### Per-module suite structure

Suite configuration is **distributed across modules**: each test module has a
`tests/<Module>/suites.php` returning an array of `SuiteConfig`. The root
`testo.php` merges them via `array_merge(require ...)`:

```php
// testo.php
suites: \array_merge(
    require __DIR__ . '/tests/Worker/suites.php',
    require __DIR__ . '/tests/KeyValue/suites.php',
),
```

```php
// tests/Worker/suites.php
return [
    new SuiteConfig(
        name: 'Worker/Unit',
        location: new FinderConfig(include: [__DIR__ . '/Unit']),
    ),
];
```

**Add tests for a new module:** create `tests/<Module>/suites.php` (suites point
at sub-directories, e.g. `Unit/`) and add a `require` line to the `array_merge`
in the root `testo.php`.

### Layout and namespaces

```
tests/
  <Module>/
    suites.php              # module suite definition (not a test, not scanned)
    Unit/                   # suite sub-directory ŌåÆ SuiteConfig.location
      FooTest.php           # namespace RoadRunner\Tests\<Module>\Unit
  Testo/                    # shared test harness (see below)
  Acceptance/               # acceptance tests against the real rr binary (see below)
  Legacy/                   # see below
```

A file's namespace must match its PSR-4 path (`RoadRunner\Tests\ => tests/`).
When you move a test into a sub-directory, update its namespace too.

Currently active: `Worker/Unit`, `KeyValue/Unit`.

### tests/Acceptance

End-to-end tests that boot the **real RoadRunner binary** and exercise the SDK
against it. The binary lives at the project root as `/rr` (gitignored).

- `tests/Acceptance/rr.yaml` ŌĆö RoadRunner config (`version: "3"`): `rpc` on
  `tcp://127.0.0.1:6001`, `server` running `php worker.php` over `pipes`, and
  the `http` plugin on `127.0.0.1:8080` with a single worker.
- `tests/Acceptance/worker.php` ŌĆö a PSR-7 HTTP worker built on the SDK
  (`RoadRunner\Sdk\Worker\Worker::create()` + `RoadRunner\Sdk\Http\PSR7Worker`,
  using `nyholm/psr7` for PSR-17).

The config is accepted by the new binary and the `http` plugin boots, **but an
end-to-end run does not work yet**: `Worker::create()` still resolves the
goridge relay (`Spiral\Goridge\...`), which is no longer installed ŌĆö this is
gated on the worker-transport rewrite. Once the transport is ported, add a
`tests/Acceptance/suites.php`, wire it into the root `testo.php`, and add a test
that starts the server via the `tests/Testo` harness and hits
`http://127.0.0.1:8080`.

### tests/Testo

Shared test harness (namespace `RoadRunner\Tests\Testo`). `RoadRunner.php`
manages the `rr` binary for acceptance tests: it runs `rr serve -c <config> -w
<workDir>` via `symfony/process`, locates the binary in the project root
(`rr.exe` on Windows, `rr` otherwise), resolves the config to an absolute path
(the new binary resolves a relative `-c` against `-w`), waits for readiness
(TCP probe or startup marker), and exposes `stop()` / `isRunning()` /
`output()`.

### tests/Legacy

`tests/Legacy/` holds the original package tests on **PHPUnit/Mockery**, not yet
ported to Testo (they mock goridge interfaces and test the not-yet-rewritten
transport). They are **not** wired into any suite and are not run by Testo.
Namespace ŌĆö `RoadRunner\Tests\Legacy\<Module>\...`. As each module is rewritten
for gRPC, its tests are ported to Testo, placed under `tests/<Module>/<Suite>/`,
and added to the module `suites.php`.

### Testo cheat sheet (differences from PHPUnit)

- A test class does **not** extend a base class. `#[Test]` on the class ŌĆö all
  public methods become tests.
- Lifecycle: `#[BeforeTest]` / `#[AfterTest]` / `#[BeforeClass]` /
  `#[AfterClass]` (instead of `setUp` / `tearDown`).
- Data: `#[DataProvider('method')]` (public static, returns `iterable`) or
  `#[DataSet([...], 'label')]` (repeatable).
- Asserts: `Assert::same($actual, $expected)` ŌĆö **order is `actual, expected`**
  (reverse of PHPUnit!), `Assert::equals/true/false/null/count/contains/instanceOf`.
- Exceptions: `Expect::exception(X::class)->withMessageContaining('...')`
  **before** Act; the method returns `never`.
- Skip: `throw new \Testo\Core\Exception\SkipTest('reason')` from the test body.
- Mocks: Testo ships none ŌĆö use **hand-written fakes** (no mocking library
  pulled in).
- `AssertPlugin` (registers `Assert` / `Expect`) is enabled automatically.
